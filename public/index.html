Mazes

This a service that gives you a maze, and you get to try to solve it. If you get it right, you get another slightly harder maze. If you get it wrong, you have to start over.

The format of a maze is a multiline string representing a 2D maze. Each character of the string represents the state of the maze at the point:

  • ' ' - An open space that you can pass through
  • 'W' - A wall that you cannot pass through
  • 'S' - The starting location, also an open space
  • 'E' - An exit to the maze, there can be multiple exits (but you only need to reach one)

The format of a solution is a string containing a series of directional moves:

  • 'U' - move up one square
  • 'D' - move down one square
  • 'L' - move left one square
  • 'R' - move right one square

When you request a maze, you will get back a JSON object containing two fields: maze and solve_url. maze is the string representation of the maze. solve_url is where you will post your solution to get the next maze.

When you attempt solve a maze, you will get back a JSON object containing two fields: result and either message or next_url. result will be either "completed" or "failure". If the maze is "completed", then next_url will contain the URL to get the next maze. If that was the last maze, instead of a URL, you will get back a message. If the maze is "failed", then message will be a message telling you why your solution did not complete the maze.

The format for requesting a maze is to make a GET request to /levels/:name?password=:password. The name for the first level is START and it has no password. When you receive a solve_url or a next_url it will already contain the password in the URL.

The format for attempting to solve a maze is to make a PUT request to the solve_url, appending your solution as an additional param called solution, either as a query param or in the body. For example, you could make a PUT to /levels/:name/solve?password=:password&solution=LLUDR

Good luck!
