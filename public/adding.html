Adding

Ziyad sucks at math so he needs someone to help him add stuff.

The format of the input is a multiline string representing a sequence of integers to add.

Each line in the string is a single integer to add.

When you request a sequence of numbers to add, you will get back a JSON object containing two fields: input and solve_url. input is the string representation of the sequence of numbers to add. solve_url is where you will post your solution to get the next sequence of numbers to add.

When you attempt solve a sequence of numbers to add, you will get back a JSON object containing two fields: result and either message or next_url. result will be either "completed" or "failure". If the sequence of numbers to add is "completed", then next_url will contain the URL to get the next sequence of numbers to add. If that was the last sequence of numbers to add, instead of a URL, you will get back a message. If the sequence of numbers to add is "failed", then message will be a message telling you why your solution did not complete the sequence of numbers to add.

The format for requesting a sequence of numbers to add is to make a GET request to /levels/:name/:stage/?password=:password. The name for the first level is adding and it has no stage or password. When you receive a solve_url or a next_url it will already contain the password in the URL.

The format for attempting to solve a sequence of numbers to add is to make a PUT request to the solve_url, appending your solution as an additional param called solution, either as a query param or in the body. For example, you could make a PUT to /levels/:name/:stage/solve?password=:password&solution=:solution

Good luck!
