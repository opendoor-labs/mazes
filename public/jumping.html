Jumping

Ziyad wants to know how good you are at long jumping.

The format of the input is a multiline string representing the distances that you can jump forward from a given index.

The solution you should provide is the sequence of jump distances (starting from the first position) that you must jump in order to make it to the end of the long jump pit (last index).

When you request a jumping distance path, you will get back a JSON object containing two fields: input and solve_url. input is the string representation of the jumping distance path. solve_url is where you will post your solution to get the next jumping distance path.

When you attempt solve a jumping distance path, you will get back a JSON object containing two fields: result and either message or next_url. result will be either "completed" or "failure". If the jumping distance path is "completed", then next_url will contain the URL to get the next jumping distance path. If that was the last jumping distance path, instead of a URL, you will get back a message. If the jumping distance path is "failed", then message will be a message telling you why your solution did not complete the jumping distance path.

The format for requesting a jumping distance path is to make a GET request to /levels/:name/:stage. The first stage is jumping.

The format for attempting to solve a jumping distance path is to make a PUT request to the solve_url, appending your solution as an additional param called solution, either as a query param or in the body as well as a secret token. For example, you could make a PUT to /levels/:name/:stage/solve?solution=:solution&token=:token

Good luck!
